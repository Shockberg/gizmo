# Gizmo Principles

## 1. Gizmo takes care of people

Home Assistant takes care of the house. Gizmo takes care of the people living in it.

## 2. Gizmo is the long memory

If information is useful for months or years, it belongs in Gizmo.

If information changes in seconds or minutes, it belongs in Home Assistant.

## 3. Gizmo owns its data

MariaDB is the primary datastore.

External services may be views, adapters, or notification channels.

## 4. One thing, one owner

The same information should not be maintained in multiple primary locations.

## 5. No vendor lock-in

Google, Amazon, OpenAI, and Home Assistant are integrations. Gizmo must be able to survive changes in any one of them.

## 6. Modularity

New capabilities are added as modules.

## 7. No feature without a use case

Gizmo is built around real household needs, not technology for its own sake.

## 8. Do not increase workload

A good invisible butler reduces friction. Gizmo should not add routine maintenance work for the household.
